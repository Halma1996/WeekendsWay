# WeekendsWay (Qt 6.8 + CMake + QML)

Pet-project на **Qt / QML** для быстрого открытия часто используемых маршрутов и мест.

Проект задумывался как Lean MVP для практики:
- **QML / Qt Quick Controls**
- **C++ ↔ QML интеграции**
- **сохранения данных в JSON**
- **сборки Android APK**
- **работы с внешним навигатором на Android и Desktop**

---

## Возможности

Приложение содержит 3 основных раздела:
- **Домой**
- **К друзьям**
- **Кушать**

Внутри разделов можно хранить списки адресов и мест, а затем быстро:
- открыть маршрут,
- открыть поиск места,
- использовать сохранённые данные повторно без ручного ввода.

---

## Технологии

- **Qt 6.8.3**
- **CMake**
- **QML / Qt Quick / Qt Quick Controls 2**
- **C++17**
- **JSON** для локального хранения данных

---

## Платформы

### Android
На Android приложение собирается в APK и открывает внешнюю навигацию через Android-механизмы запуска приложений.

### Desktop
На Desktop используется fallback через открытие Google Maps / web URL в браузере.

---

## Структура проекта

- `qml/` — интерфейс на QML
- `src/` — C++ логика
- `icons/` — иконки
- `android/` — Android-специфичные файлы, если используются
- `build/` — артефакты сборки (в репозиторий не добавляются)

Основные C++ классы:
- `Navigator` — открытие маршрутов и поисковых запросов во внешнем навигаторе
- `Storage` — сохранение и чтение пользовательских данных в JSON

---

## Хранение данных

Приложение использует:

`QStandardPaths::AppDataLocation`

Данные сохраняются локально в JSON-файлы.

На страницах списков есть кнопка:

**«Открыть данные (AppData)»**

Она помогает быстро открыть папку с сохранёнными данными.

---

## Navigator

Из QML доступны методы:

### `Navigator.openRoute(destination, travelMode)`
Открывает маршрут до указанного адреса или места.

Поддерживаемые `travelMode`:
- `"walking"`
- `"driving"`
- `"transit"`
- `"bicycling"`

### `Navigator.openSearch(query)`
Открывает поиск места или запроса в навигаторе / картах.

## Логика работы по платформам

### Android
Используется запуск внешнего приложения навигации.

### Desktop
Используется web fallback через браузер.

Если на Android нет подходящего приложения-обработчика, используется fallback на web-открытие.
